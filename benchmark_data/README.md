# Benchmark Data

This directory stores evaluation datasets for the SMalRe malware analysis benchmark,
compatible with the **CyberSOCEval** framework.

---

## Expected Files

| File | Description |
|------|-------------|
| `malware_analysis_questions.json` | Multiple-choice questions derived from Hybrid Analysis sandbox reports |
| `responses.json` | Model responses (generated by `eval_suite.py`) |
| `judge_responses.json` | Teacher LLM judgements (when using LLM-as-judge mode) |
| `stats.json` | Evaluation statistics (accuracy, Jaccard, breakdowns) |

---

## Question Format

Each entry in `malware_analysis_questions.json` should follow this schema:

```json
{
  "id": "q_001",
  "question": "Which of the following best describes the primary persistence mechanism used by the sample?",
  "context": "<sandbox detonation report JSON or summary text>",
  "options": {
    "A": "Registry Run key modification",
    "B": "Scheduled task creation",
    "C": "DLL side-loading",
    "D": "Boot sector modification",
    "E": "Service installation"
  },
  "answer": ["B", "E"],
  "difficulty": "Medium",
  "topic": "Persistence Techniques",
  "malware_type": "RAT"
}
```

### Field Descriptions

- **id** – Unique question identifier.
- **question** – The question text.
- **context** – Optional JSON string of the sandbox detonation report (or a summary).
  Use `--truncate-input` to filter large reports to essential fields.
- **options** – Object mapping option labels (A–J) to answer text.
- **answer** – Array of correct option labels (multi-select supported).
- **difficulty** – `"Easy"`, `"Medium"`, or `"Hard"`.
- **topic** – Reasoning category, e.g.:
  - `"Persistence Techniques"`
  - `"Evasion Techniques"`
  - `"Process Injection"`
  - `"MITRE ATT&CK Mapping"`
  - `"Network IOC Extraction"`
  - `"Risk Assessment"`
- **malware_type** – One of: `"EDR/AV Killer"`, `"Ransomware"`, `"RAT"`,
  `"Infostealer"`, `"UM Unhooking"`, `"Other"`.

## Running CyberSOCEval Malware Analysis Benchmark

```bash
python3 -m CybersecurityBenchmarks.benchmark.run \
  --benchmark=malware_analysis \
  --prompt-path="benchmark_data/malware_analysis_questions.json" \
  --response-path="benchmark_data/responses.json" \
  --judge-response-path="benchmark_data/judge_responses.json" \
  --stat-path="benchmark_data/stats.json" \
  --llm-under-test=<PROVIDER>::<MODEL>::<API_KEY> \
  --truncate-input
```

Replace `<PROVIDER>::<MODEL>::<API_KEY>` with your SLM configuration, e.g.:

- `OPENAI::gpt-4o::<key>` (teacher evaluation)
- `TOGETHER::meta-llama/Llama-3-8B::<key>`
- `TOGETHER::Qwen/Qwen2-7B-Instruct::<key>`

## Sourcing Benchmark Data

Test cases in this benchmark are derived from **Hybrid Analysis** / CrowdStrike
Falcon® Sandbox report data.  To obtain data:

1. Create a free account at [hybrid-analysis.com](https://hybrid-analysis.com)
2. Download JSON-format detonation reports for samples from the five target families
3. Use `report_parser.py` to convert raw reports to the question-ready format
4. Use `data_filter.py` to apply the `--truncate-input` pre-filtering

---

## Metrics

| Metric | Description |
|--------|-------------|
| Accuracy | Fraction of questions with an exact-match answer set |
| Avg Jaccard | Average Jaccard similarity (partial credit) across all questions |

The `stats.json` output also includes breakdowns by **difficulty**, **topic**,
and **malware type**.
